Node counts are actual measured results from C++ program output (globals.cpp). Race animation reconstructs the optimal solution path via BFS in JavaScript. Step mode g(n)/h(n) values are computed live from the board state at each step — g(n) is the step index along the BFS-optimal path, h(n) is recomputed from that exact board configuration. These values may differ from what your C++ search actually expanded (the search explores many non-optimal nodes), but they correctly describe the cost function at each point on the solution path itself.
